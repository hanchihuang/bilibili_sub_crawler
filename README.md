# B 站 UP 主字幕爬取工具

本地网页工具：输入 UP 主 UID，自动爬取该 UP 主**全部视频**的**全部字幕**，支持在页面查看和导出 ZIP。

## 重要提醒（务必先看）

> [!WARNING]
> **使用 `vCaptions` 自动抓取字幕时，程序会自动执行“下载 -> 下拉 -> 复制”。这里有一个需要特别注意的现象：首次点击“复制”有可能失败。**
>
> **如果你发现第一次没有复制成功，需要人工帮忙点一次“复制”按钮。只要这次手动复制成功，后续通常就不需要再干预，程序一般会继续稳定自动复制。**
>
> **这不是必然发生的问题，很多时候也可能完全不会出现。这里专门写出来，是为了提醒你一旦遇到，不要误以为程序卡死。优先检查是不是首次复制失败，并手动补点一次“复制”。**

## 使用步骤

1. 安装依赖（在项目目录下）：
   ```bash
   pip install -r requirements.txt
   ```

2. 启动服务：
   ```bash
   python app.py
   ```

3. 浏览器打开：**http://127.0.0.1:5000**

4. 在输入框填写 **UP 主 UID**（博主号）。UID 是数字，可在 UP 主空间页地址里看到，例如：
   - 打开 `https://space.bilibili.com/123456`，则 UID 为 `123456`。

5. 如果你走的是 `wait_vcaptions.py` 这条自动抓取流程，**运行到中途时，需要手动安装 `vCaptions` 扩展**。不要关掉浏览器，程序会停下来等你安装完成。

6. 点击「开始爬取」，等待完成后在页面查看字幕，或点击「导出 ZIP」下载全部字幕文本。

7. 如果你使用的是 `vCaptions` 自动抓取流程，看到程序已经自动点击下载并下拉到复制位置后，要留意**第一次复制是否成功**：
   - 若成功，后续一般无需人工干预。
   - 若失败，**请人工手动点击一次“复制”**；这一步成功后，后面通常会一直复制成功。
   - 这类情况**不一定会发生**，只是额外提醒。

## 中途安装 vCaptions 的步骤

> [!IMPORTANT]
> **使用 `wait_vcaptions.py` 时，程序运行到中途会暂停，并要求你在 Chrome 里手动安装 `vCaptions`。这是正常步骤，不是卡住。**

安装步骤如下：

1. 运行：
   ```bash
   python wait_vcaptions.py
   ```

2. 程序会自动打开 Chrome，并停在扩展管理页 `chrome://extensions/`。

3. 在扩展页面右上角打开**开发者模式**。

4. 点击「加载已解压的扩展程序」。

5. 选择本项目里的扩展目录：
   ```text
   vcaptions_extension
   ```

6. 确认 `vCaptions` 已成功显示在扩展列表里并处于启用状态。

7. 回到终端，按回车继续。

8. 程序随后还会提示你先在浏览器里登录 B 站；登录完成后，再回终端按回车，程序才会继续批量抓取字幕。

## 说明

- **未登录**时，部分视频的字幕接口可能返回空（B 站对部分稿件要求登录后才返回字幕）。若发现大量无字幕，可考虑在浏览器登录 B 站后，把 Cookie 中的 `SESSDATA` 等提供给本工具（当前版本未做 Cookie 配置，可自行改代码加请求头）。
- 请求已做简单限速，避免触发风控；视频较多时爬取会稍慢，属正常现象。

## 项目结构

- `app.py`：Flask 服务，提供网页与 `/api/crawl`、`/api/export`
- `bilibili_api.py`：B 站接口封装（视频列表、分 P、字幕列表、字幕内容）
- `wait_vcaptions.py`：等待 vCaptions 扩展安装后批量爬取（需手动安装扩展）
- `templates/index.html`：前端页面
- `requirements.txt`：Python 依赖

## wait_vcaptions.py 使用方法

```bash
# 启动后会：
# 1. 打开 Chrome 浏览器并加载 vCaptions 扩展
# 2. 运行到中途会暂停，提示用户在浏览器中手动安装扩展
# 3. 安装完成并回车后，继续登录 B 站
# 4. 然后自动批量爬取指定 UP 主的所有视频字幕
# 5. 自动执行“下载 -> 下拉 -> 复制”；若首次复制失败，需人工手动点一次“复制”

python wait_vcaptions.py

# 或者使用 API 模式（不需要 vCaptions 扩展）
python wait_vcaptions.py --api <UP主UID>
```
